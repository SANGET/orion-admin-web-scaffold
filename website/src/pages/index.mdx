## 国际化

### 原理

@deer-ui/admin-scaffold 的国际化策略为：通过用户浏览器默认语言，以及右上角提供的选择器确定目标语言，获取指定目录 json 文件，包装成符合 @deer-ui 国际化策略的方式实现。

例如

浏览者浏览器语言为中文，则获取项目的 `/public/i18n/zh-CN.json` 文件，或者切换到英文时，获取 `/public/i18n/en-US.json`

1. 国际化文件以 json 文件为主，默认在项目的 `/public/i18n/` 目录下，以 `${lang}.json` 命名存放。
2. 如果自定义了国际化目录，则需要通过 `AdminWebScaffold.setI18nUrl` 接口设置国际化文件目录路径。
3. 脚手架尝试获取国际化目录下的国际化配置文件，并且导入项目当中，配合 @deer-ui 的国际化策略。

### 国际化选项配置

```js
import zhCN from '@deer-ui/admin-scaffold/i18n/zh-CN';
import enUS from '@deer-ui/admin-scaffold/i18n/en-US';

<AdminWebScaffold {...props}
  // 内置的国际化文件
  i18nMapper={{
    'zh-CN': zhCN,
    'en-US': enUS,
  }}
  // 内置的国际化文件
  i18nConfig={{
    'zh-CN': '中文',
    'en-US': 'English',
  }} />
```

### 设置国际化配置路径

```js
import { AdminWebScaffold } from '@deer-ui/admin-scaffold'

AdminWebScaffold.setI18nUrl('/public/i18n/');
```

### 国际化文件事例

`/public/i18n/zh-CN.json` 文件

```json
{
  "查询": "查询",
  "Status": "状态",
  "Nickname": "昵称",
  "Phone": "电话"
}
```
